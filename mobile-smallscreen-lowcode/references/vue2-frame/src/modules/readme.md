# modules目录说明

## 说明：

为了方便开发人员，M8跨端框架中封装了一些固定业务功能模块，通过路由页面跳转可以访问。

注意：

- 项目产品开发的业务模块**入口**始终在pages目录下，不要放在modules目录下。

- pages下的模块始终支持router.js中配置为分包模块。


## 什么样的业务模块？

一些固定的业务功能的模块，并且这些模块也存在一定的路由跳转功能。产品项目也可以将一些固定的业务功能模块放在modules目录下，

比如：

  - 通用的跨端选人模块

  - 通用的平台小程序用户绑定注册模块

  - m8showcase示例模块

  - m8demo示例模块
  
  - 产品项目自定义的模块


## 集成到了框架的什么地方？

目录结构如下

```
ejs.m8.mobileframe
    |- src          
    |   |- modules          // 存放框架集成的模块的地方
    |   |   |- m8showcase   // 对应的M8UI组件示例模块
    |   |   |- xxx          // 对应的模块资源
    |   |- pages            // 存放所有业务相关代码
```
## modules分包

modules目录以分包的形式提供，框架读取modules/**下的目录中的router.js/mock.js/store.js文件。会与pages目录下的模块注册方式一致，同时自动以分包的方式注册到src/pages.json中的分包节点中。
