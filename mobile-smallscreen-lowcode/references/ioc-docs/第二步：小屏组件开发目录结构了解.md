==小白应该知道我们的小屏组件开发目录在哪里，目录的下每个文件 代表什么意思？？？不然当遇到问题很难排查定位出问题所在。==
```

│   ├── js/
│   │   ├── base.js         # 组件基础信息

│   ├── plugin/
```

## 组件结构说明

### 1. js/base.js - 组件基础信息
**作用：** 定义组件的基本元数据，包括组件名称、模块标识和版本号。
```javascript
export default {
    name: 'XXX',                  // 组件中文名称（在低码平台显示）
    module_name: 'XXXX',     // 组件模块名（必须与文件夹名完全一致）
    version: '0.1.0'                 // 版本号（遵循语义化版本规范）
};
```

**注意事项：**
-  `module_name` 必须使用**大驼峰命名**（PascalCase），如：`ShoppingCart`、`ProductList`
-  `module_name` 必须与组件文件夹名**完全一致**，否则组件无法正常注册
-  `name` 是组件的中文名称，会显示在低码平台的组件列表中
-  `version` 建议使用语义化版本号（major.minor.patch）

### 2. js/config.js - 组件配置项
**作用：** 定义组件在低码平台中可配置的属性和事件，是组件灵活性的核心。

**配置项分为两大类：**

#### options - 组件属性配置

```javascript
{
    name: 'options',
    value: [
        boxOptions,              // 通用容器样式（必须）
        {
            displayName: '布局设置',  // 配置组名称
            name: 'layout',          // 配置组标识
            value: [
                {
                    displayName: '布局方向',  // 配置项显示名
                    name: 'direction',       // 配置项标识
                    value: 'horizontal',     // 默认值
                    type: 'select',          // 配置项类型
                    options: [...]           // 选项（select 类型需要）
                }
            ]
        }
    ]
}
```

#### interaction - 交互配置

```javascript
{
    name: 'interaction',
    displayName: '交互',
    value: [
        {
            name: 'callback',        // 回调参数
            displayName: '回调参数',
            type: 'array',
            dynamic: true,
            value: [],
            template: {...}
        },
        {
            name: 'event',           // 事件配置
            displayName: '事件（新）',
            type: 'array',
            value: [
                {
                    name: 'onMounted',
                    displayName: 'onMounted事件',
                    dynamic: true,
                    type: 'array',
                    value: [],
                    template: {...}
                }
            ]
        }
    ]
}
```
### css/index.scss - 组件样式
**作用：** 定义组件的样式，使用 SCSS 预处理器语法。

**推荐使用 BEM 命名规范：**

```scss
// Block（块）
.shopping-cart {
    display: flex;
    flex-direction: column;
    
    // Element（元素）
    &__header {
        padding: 16px;
        font-size: 18px;
    }
    
    &__item {
        display: flex;
        padding: 12px;
        
        // Modifier（修饰符）
        &--disabled {
            opacity: 0.5;
        }
    }
    
    &__price {
        color: #ee0a24;
        font-weight: 600;
    }
}
```

**样式编写建议：**

- 使用 BEM 命名规范，避免样式冲突
- 使用 SCSS 嵌套，提高代码可读性
- 使用变量定义颜色、字号等常量
- 考虑移动端适配（使用 rem、vw 等单位）
- 添加必要的注释说明

### index.vue - 组件主文件

**作用：** 组件的核心文件，包含模板、逻辑和样式引用。

**必须包含的部分：**

```vue
<template>
    <div class="epoint-component" :style="[boxOptions, boxOptions.boxmodel, positionSetting]">
        <!-- 组件 UI 结构 -->
    </div>
</template>

<script>
// 必须引入
import '@boot';                          // 启动配置
import mockData from './mock/data';      // Mock 数据
import defaultConfig from './js/config'; // 配置项
import base from './js/base';            // 基础信息
import eventMixin from './plugin/eventgenerate'; // 事件混入

export default {
    name: base.module_name,              // 必须使用 base 中的名称
    mixins: [eventMixin],                // 必须混入事件
    
    props: {
        data: {                          // 数据源
            type: Array,
            default: () => mockData.data
        },
        config: {                        // 配置项
            type: Object
        },
        cdata: Object                    // 父组件数据
    },
    
    data() {
        return {
            // 组件内部状态
        };
    },
    
    computed: {
        //处理容器样式
        boxOptions() {
            const { boxOptions } = this.config.options;
            const boxmodel = boxOptions.boxmodel;
            
            Object.keys(boxmodel).forEach((key) => {
                if (!isNaN(Number(boxmodel[key]))) {
                    boxmodel[key] = boxmodel[key] + 'px';
                }
            });
            
            return boxOptions;
        },
        
        // 获取配置项
        headerConfig() {
            return this.config?.options?.headerConfig || {};
        }
    },
    
    mounted() {
        //  必须：触发 onMounted 事件
        this.eventGenerate('onMounted', { data: this.myData });
    },
    
    methods: {
        // 组件方法
    },
    
    // 必须：导出配置和数据
    _getConfig: () => defaultConfig,
    _getMockData: () => mockData
};
</script>

<style lang="scss" scoped>
@import './css/index.scss';
</style>
```

**💡 关键点说明：**

**1. 根元素必须包含的样式绑定：**

```vue
<div :style="[boxOptions, boxOptions.boxmodel, positionSetting]">
```

**2. 数据优先级处理：**

```javascript
computed: {
    myData() {
        // 优先使用父组件传递的数据
        const pdata = this.cdata?.pdata;
        if (pdata && Array.isArray(pdata) && pdata.length > 0) {
            return pdata;
        }
        // 否则使用 props.data 或 mock 数据
        return this.data || mockData.data || [];
    }
}
```

**3. 事件触发方式：**

```javascript
methods: {
    handleClick(item, index) {
        this.eventGenerate('onClick', { item, index });
    }
}
```

### index.js - 组件注册入口

**作用：** 组件的注册入口，用于在 Vue 中全局注册组件。

**文件内容：**

```javascript
/**
 *  组件注册入口
 * 
 *  此文件由脚手架自动生成，请勿手动修改
 */

import component from './index.vue';
import base from './js/base.js';
import config from './js/config.js';

// 组件注册
component.install = function(Vue) {
    Vue.component(base.module_name, component);
};

// 导出组件配置
component.config = config;
component.base = base;

export default component;
```

**注意事项：**

- 此文件通常由 `npm run ioc create` 命令自动生成
- 一般情况下不需要手动修改
- 确保正确引入了 `index.vue`、`base.js` 和 `config.js`

###  plugin/eventgenerate/index.js - 事件生成插件


**作用：** 提供 `eventGenerate` 方法，用于触发低码平台配置的事件。

**文件内容：**

```javascript
/**
 * 事件生成插件
 * 
 * 提供 eventGenerate 方法，用于触发低码平台配置的事件
 */

export default {
    methods: {
        /**
         * 触发事件
         * @param {string} eventName - 事件名称（必须与 config.js 中 interaction.event 的 name 一致）
         * @param {any} params - 传递给低码平台的参数对象
         */
        eventGenerate(eventName, params) {
            // 获取事件配置
            const events = this.config?.interaction?.event || [];
            const eventConfig = events.find(e => e.name === eventName);
            
            if (!eventConfig || !eventConfig.value || eventConfig.value.length === 0) {
                return;
            }
            
            // 执行配置的动作
            eventConfig.value.forEach(action => {
                try {
                    // 创建执行上下文
                    const e = params;
                    // eslint-disable-next-line no-eval
                    eval(action.value);
                } catch (error) {
                    console.error('[' + eventName + '] 事件执行失败:', error);
                }
            });
        }
    }
};
```

**使用方式：**

```javascript
// 在 index.vue 中
export default {
    mixins: [eventMixin],  // 混入事件插件
    
    mounted() {
        // 触发 onMounted 事件
        this.eventGenerate('onMounted', { data: this.myData });
    },
    
    methods: {
        handleClick(item) {
            // 触发自定义事件
            this.eventGenerate('onClick', { item });
        }
    }
};
```

### 配置项类型详解

| 类型          | 说明       | 示例               | 使用场景 |
| ------------- | ---------- | ------------------ | -------- |
| `text`        | 文本输入框 | 标题、描述等       | 文字内容 |
| `number`      | 数字输入框 | 字号、间距等       | 数值配置 |
| `boolean`     | 开关       | 是否显示、是否启用 | 开关控制 |
| `color`       | 颜色选择器 | 背景色、文字颜色   | 颜色配置 |
| `select`      | 下拉选择   | 布局方向、对齐方式 | 枚举选择 |
| `array`       | 数组配置   | 动态列表项         | 列表数据 |
| `boxmodel`    | 盒模型     | 内外边距           | 间距配置 |
| `uploadimage` | 图片上传   | 图标、背景图       | 图片资源 |
